# MemoryNexus Engine

MemoryNexus Engine is the internal long-term feedback core. It is not an app,
role, chatbot, UI, or generic memory API.

The Engine converts experience and performance traces into memory structures,
feedback loops, growth models, sleep consolidation results, and next plans.

```text
Trace
-> MemoryAtom
-> CognitiveScene
-> FeedbackLoop
-> GrowthModel
-> SleepCycle
-> PracticePlan / DreamCandidate
```

External callers should not manipulate Engine internals directly. They should
enter through the Surface Gateway.

## Engine Responsibilities

- Maintain `CognitiveSpace` ownership and provenance.
- Partition long-running domains with `Namespace`.
- Record interaction, practice, reflection, and planning evidence as `Trace`.
- Extract minimal cognitive signals as `MemoryAtom`.
- Consolidate atoms into `CognitiveScene`.
- Track practice and review loops with `FeedbackLoop`.
- Maintain namespace-specific `GrowthModel`.
- Run offline `SleepCycle` consolidation.
- Generate `PracticePlan` / `DreamCandidate` records.
- Apply `Lens` as an interpretation strategy when reflection or planning needs a
  perspective.

## Core Objects

### CognitiveSpace

`CognitiveSpace` remains the ownership and permission boundary. Memory belongs
to a Space, not to an agent, app, adapter, or namespace.

### Namespace

`Namespace` is a domain partition inside a Space.

Examples:

```text
child.chinese.dictation
child.english.spelling
personal.thoughts
project.memorynexus
skill.piano
skill.chess
```

Namespace answers:

```text
Which long-running domain does this evidence belong to?
```

It does not answer:

```text
Who owns this data?
```

### Trace

`Trace` is one interaction or execution with input/output summaries, runtime
metrics, generated object links, and optional user feedback.

Trace is the evidence layer for learning whether feedback and plans are useful.

### MemoryAtom

`MemoryAtom` is the smallest traceable cognitive signal extracted from Trace,
Capture, or Memory material.

### CognitiveScene

`CognitiveScene` is a long-running theme, problem field, or practice scene
formed by related atoms, reflections, concepts, beliefs, contradictions, and
growth signals.

### FeedbackLoop

`FeedbackLoop` captures:

```text
goal -> task -> attempt -> evaluation -> feedback -> next task
```

It is the operational loop where practice and feedback become observable.

### GrowthModel

`GrowthModel` is the namespace-specific long-term growth picture.

It may include:

- strengths;
- weaknesses;
- recurring patterns;
- current stage;
- recommended focus;
- evidence IDs;
- stability or mastery signals.

GrowthModel is not a generic user profile. It is domain-specific and evidence
backed.

### SleepCycle

`SleepCycle` is an offline consolidation period such as daily, weekly, or manual.
It reads Trace / Memory / FeedbackLoop evidence and produces consolidation
outputs.

### PracticePlan / DreamCandidate

`PracticePlan` is the product-facing next plan. `DreamCandidate` is the internal
candidate generated by Dreaming. A candidate becomes useful only after later Wake
traces show whether it helped.

### Lens

`Lens` is an interpretation strategy. It is not an agent persona or identity.
Surfaces can use lenses for review, reflection, and planning, but adapters should
not present Lens as role-playing unless a product design explicitly allows it.

## Engine Non-Goals

- Do not become a model runtime.
- Do not become a generic memory cloud.
- Do not expose raw Engine objects to upstream apps.
- Do not own user identity through agents.
- Do not do OCR, broad curriculum, or complex UI in the first vertical.
- Do not run deep consolidation on every foreground request.

## Engine Loop

```text
Wake:
  SurfaceRequest
  -> validate actor / namespace / payload
  -> quick response or immediate feedback
  -> Trace
  -> optional Engine Event

Sleep:
  Engine Event / manual trigger
  -> aggregate Trace evidence
  -> update GrowthModel
  -> generate PracticePlan / DreamCandidate

Next Wake:
  user attempts next task
  -> Trace
  -> effectiveness evaluation
```
